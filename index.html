Cascading Style Sheets (CSS)

CSS (Каскадные Таблицы Стилей) — это язык таблицы стилей. Он позволяет применять стили к элементам в документах HTML

        .my-element {
            font-size: 18px;
            line-height: 1.5;
            color: crimson;
        }

CSS: правила

        .my-element {
            font-size: 18px;
            line-height: 1.5;
            color: crimson;
        }

CSS: правила

        .my-element { // Селектор (Selector)
            font-size: 18px;
            line-height: 1.5;
            color: crimson;
        }

CSS: правила

        .my-element {
            font-size: 18px;  // Объявление (Declaration)
            line-height: 1.5;
            color: crimson;
        }

CSS: правила

        .my-element {
            font-size: 18px;  // Свойство (Property) 
            line-height: 1.5; // Свойство (Property) 
            color: crimson; // Свойство (Property) 
        }

CSS: правила

        .my-element {
            font-size: 18px;  // Значение свойства (Property value)
            line-height: 1.5; // Значение свойства (Property value)
            color: crimson; // Значение свойства (Property value)
        }

CSS: правила

          .my-element {

CSS: правила

              font-size: 18px ;

  1. Набор правил должен быть обёрнут в фигурные скобки - {}

  3. Объвление заканчивается точкой с запятой - ;

Псевдоклассы

Псевдокласс — это селектор, который выбирает элементы, находящиеся в специфическом состоянии

:pseudo-class-name

Псевдоэлементы

Псевдоэлемент — это элементы, которых не существует в HTML-разметке. Оведут себя сходным образом с псевоклассами, однако действуют так, как если бы вы добавили в разметку целый новый HTML-элемент, а не применили класс к существующим элементам

::pseudo-element-name

CSS селекторы: комбинирование и группировка

  1. Группировка: .element1, .element2
  2. Объединение: .class1.class2
  3. Потомки: .element1 .element2
  4. Непосредственно вложенные: .element1 > .element2
  5. Смежные: .element1 + .element2
  6. Последующие: .element1 ~ .element2

CSS селекторы: группировка

Селекторы записываются через запятую

До

            .element1 { color: red; }
            .element2 { color: red; }
            .element3 { color: red; }

После

              .element1,
              .element2,
              .element3 {
                  color: red;
             }

CSS селекторы: объединение

Селекторы записываются слитно 
        .class1.class2 {
            color: red;
        }

CSS селекторы: потомки

Селекторы записываются последовательно через пробел 
        .block h2 {
            color: red;
        }

CSS селекторы: непосредственно вложенные

Селекторы разделяются знаком > 
        .block > h2 {
            color: red;
        }

CSS селекторы: cмежные

Селекторы разделяются знаком + 
        .block1 + .block2 {
            color: red;
        }

CSS селекторы: последующие

Селекторы разделяются знаком ~ 
        .block1:hover ~ .block2 {
            color: red;
        }

CSS: специфичность, наследование, каскад

Каскад — к одному и тому же элементу может применяться несколько CSS-правил (наборов CSS-свойств). Каскадирование определяет, какие именно свойства из всех возможных источников будут применены к элементу.

Флексбокс: главная и поперечные оси

Cвойство justify-content распределяет элементы относительно главной оси

  • flex-start — элементы стоят в начале оси
  • flex-end — элементы стоят в конце оси
  • center — элементы по центру оси
  • space-between — первый элемент располагается в начале, а последний — в конце. Между крайними элементами и флекс-контейнером отступа нет

Флексбокс: главная и поперечные оси

Элементы равномерно распределены вдоль главной оси

  • space-around — расстояние между крайними элементами и флекс-контейнером равно половине отступа между соседними флекс-элементами
  • space-evenly — отступы между соседними элементами и краями флекс-контейнера равны

Флексбокс: главная и поперечные оси

Cвойство align-items распределяет элементы относительно поперечной оси

  1. stretch — элементы растягиваются вдоль всей оси (по умолчанию)
  2. flex-start — элементы идут в начале оси
  3. flex-end — элементы идут в конце оси
  4. center — элементы в центре оси
  5. baseline — элементы размещены по базовой линии

Флексбокс: главная и поперечные оси

baseline — невидимая линия, которая проходит по нижней части букв

Флексбокс: главная и поперечные оси

align-self — выравнивает отдельный элемент относительно поперечной оси

Значения align-self такие же, как у align-items

Флексбокс: главная и поперечные оси

align-content — выравнивает ряды элементов вдоль поперечной оси.

  1. flex-start
  2. flex-end
  3. center
  4. space-between
  5. space-around
  6. space-evenly
  1. stretch
  2. baseline

Отличие align-content от align-items

  • align-items выравнивает элементы внутри флекс-контейнера
  • align-content выравнивает ряды

Когда align-content задано другое значение (не stretch), оно перекрывает значения align-items и align-self

flex-wrap

            display: flex;
            flex-wrap: wrap;

flex-grow

flex-grow определяет, как элементы будут растягиваться внутри контейнера, когда есть свободное пространство

        .element-1 {
            flex-grow: 0;
       }
        .element-2 {
            flex-grow: 1;

flex-shrink

flex-shrink отвечает за гибкость и сжатие элементов, когда доступное пространство ограничено

        .element-1 {
            flex-shrink: 0;
       }
        .element-2 {
            flex-shrink: 1;

flex-basis

flex-basis задает начальный размер элемента перед распределением доступного пространства

          .element-1 {
              height: 300px;
              width: 300px;
              flex-basis: 500px;
          }
        
          .element-2 {
              height: 300px;
              width: 300px;
          }
        
          .element-3 {
              height: 300px;
              width: 300px;
          }

order

order определяет порядок отображения элементов в контейнере

        .element-1 {
            order: 2;
        }
        .element-2 {
            order: 1;

Flexbox

Свойство flex работает только для элементов, у которых значение свойства display установлено на flex

        .element {
            display: flex;
        }

Flexbox

Если элементы не помещаются в одну строку или колонку из-за ограничений ширины или высоты контейнера, используется свойство flex-wrap со значением wrap

        .element {
            display: flex;
            flex-wrap: wrap;
        }

Flexbox

Расположение элементов в HTML-файле не имеет решающего значения

        .element {
            display: flex;
            order: 2;
        }

Flexbox

Во флексах есть понятие начала (start) и конца (end), поэтому в браузерах с локалью rtl (right-to-left) все элементы будут автоматически расположены в реверсном порядке

Grid Layout

Гриды

CSS Grid Layout или просто гриды — это технология для раскладки элементов на веб-страницах.

Гриды дают возможность работать одновременно с двумя измерениями: горизонталью и вертикалью

        .element {
            display: grid;
        }

Гриды

  1. Грид-контейнер — родительский элемент, к которому применяется свойство display: grid
  2. Грид-элемент — дочерний элемент, прямой потомок грид-контейнера
  3. Грид-линия — разделительная линия, формирующая структуру грида. Может быть как вертикальной (грид-линия колонки), так и горизонтальной (грид-линия ряда). Располагается по обе стороны от колонки или ряда

Свойства грид-контейнера

  • grid-template-columns
  • grid-template-rows

grid-template-columns

Будет создано 3 колонки

        .container {
            display: grid;
            grid-template-columns: 100px auto 30%;
        }

grid-template-rows

Будет создано 3 ряда

        .container {
            display: grid;
            grid-template-rows: 200px 15vw 10rem;
        }

repeat()

Если нужны одинаковые колонки или ряды, то можно воспользоваться функцией repeat()

        .container {
            display: grid;
            grid-template-columns: repeat(3, 1fr);
        }

fr

fr (fraction) — новая единица измерения. Отвечает за свободное пространство внутри грид-контейнера

grid-template-columns: repeat(3, 1fr);

то же самое что и

grid-template-columns: 1fr 1fr 1fr;

auto-fill и auto-fit

В тех случаях, когда количество колонок или строк не известно можно установить значения auto-fill или auto-fit

        .container {
            display: grid;
            grid-template-rows: repeat(auto-fill, 100px);
        }

auto-fill

auto-fill стремится заполнить колонками всё доступное пространство, а когда элементы заканчиваются, создаёт пустые колонки, равномерно распределяя доступную область между существующими и «виртуальными» колонками

auto-fit

auto-fit схлопывает пустые колонки и отдаёт больше места под заполненные

grid-auto-columns, grid-auto-rows

Если элементов внутри грид-контейнера больше, чем может поместиться в заданные явно ряды и колонки, то их колонки и ряды можно создавать автоматически

        .container {
            display: grid;
            grid-template-rows: 100px 200px;
            grid-auto-rows: 50px;
        }

grid-auto-flow

grid-auto-flow указывает как размещать новые элементы внутри родителя

        .container {
            display: grid;
            grid-template-columns: 50px 100px 200px;
            grid-auto-columns: 50px 75px 100px;
            grid-auto-flow: columns;
        }

grid-auto-flow

  1. row — автоматически размещаемые элементы выстраиваются в ряды (значение по умолчанию)
  2. column — автоматически размещаемые элементы выстраиваются в колонки
  3. dense — браузер старается заполнить пустые ячейки в разметке, если размеры элементов позволяют. Можно сочетать с остальными значениями

grid-template-areas

  1. none — имя не задано
  2. • — пустая ячейка
  3. название — название области

grid-template-areas

         .container {
            ...
            grid-template-areas:
                "header header header"
                "content content content"
                "content content ."
                "footer footer footer";
        }

grid-template

Свойство-шорткат для свойств grid-template-rows, grid-template-columns

        .container {
          display: grid;
          grid-template: repeat(2, 1fr) / repeat(3, 100px);

row-gap, column-gap

        .container {
          display: grid;
          grid-template: repeat(2, 1fr) / repeat(3, 100px);
          row-gap: 20px; // расстояние между рядами
          column-gap: 30px; // расстояние между колонками
        }

Свойство-шорткат для свойств row-gap, column-gap

Расположение элемента внутри грид-сетки

  1. название или номер линии — порядковый номер или название конкретной линии
  2. span [число] — элемент растянется на указанное количество ячеек
  3. span [имя] — элемент будет растягиваться до следующей указанной линии
  4. auto — автоматическое растягивание элемента

  2. min-content — размер определяется по самому наименьшему элементу
  3. max-content — размер определяется по самому большому элементу
  4. minmax(min, max) — определяет диапазон размеров элемента
  5. fr — доля (fraction) доступного пространства в грид-контейнере
  6. auto — диапазон между минимумом и максимумом. В большинстве случаев это то же самое, что и minmax(min-content,max-content)

Анимация свойств

Для анимации доступны следующие свойства:

  1. gap, row-gap, column-gap
  2. grid-template-columns, grid-template-rows

Flex vs Grid

Flexbox Layout vs Grid Layout

  1. Флексы — одномерные
  2. Гриды — двухмерные

Гриды в основном используются для построения сетки всего интерфейса. Флексы же — для расположения элементов внутри сетки.

Практика

  1. Выделить в макете крупные сетки с помощью флексов
  2. Выделить в макете мелкие сетки компонентов с помощью флексов
  3. Выделить в макете грид-контейнеры
